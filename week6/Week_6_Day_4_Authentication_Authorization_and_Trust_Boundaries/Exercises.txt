ğŸ§ª Exercise 4 â€” Authentication Models (Design Thinking)

For each model, write 1 sentence describing:
What it is
One risk or downside

Models:

Username + Password - Account name for a user, and a password to allow access - downside is that if a password is too long or 
complex, the user might write it down.

API Keys - It's a unique code that identifies and authenticates an application or user when they request access to an 
API, a downside is that they authenticate the application, not usually a user.

Session Cookies - Session cookies are temporary, transient files that websites use to track a user's activity during 
a single browsing session, holding information like login status or shopping cart contents, and are automatically deleted 
when the browser is closed, good for storing state, not meant for security - authentication or authorization.

JWT (JSON Web Tokens) - A JSON object that can be included in a URL to authenticate and authorize a user - risks are 
that they can be easily decoded, or intercepted at the client side.
