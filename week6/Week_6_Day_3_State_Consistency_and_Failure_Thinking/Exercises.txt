Answer in your own words:
What does it mean for an operation to be idempotent?
Idempotency is a property where applying an operation multiple times yields the same result as applying it once, 
preventing unintended side effects like duplicates.

Why is idempotency important for APIs?
So that retries after failures don't corrupt data or double-charge customers. For example, in the previous question, the user
submitted an application, then immediately refreshed the list. If the system returns the old list, they might re-submit their
application. A dupliicate application should not be stored, so the new application would be rejected.

Is POST /applications usually idempotent? Why or why not? Yes, for the same reason: there should not be duplicate data, 
so making POST /applications idempotent would prevent duplicate requests from creating duplicate data.
